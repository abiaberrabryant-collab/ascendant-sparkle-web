# Signal Studio — Page Override

**Purpose:** A dense, calm review workspace for sales signals. It must make the source, matching reason, and approval state obvious at a glance.

- Use the project’s existing purple primary for actions and status; reserve destructive red for errors only.
- Use compact data cards, visible labels, and clear empty states. Do not use decorative animations in the queue.
- Keep the review workflow explicit: source → match reason → contact details → editable draft → approval → send from the user’s mail client.
- Every button needs a visible label, focus state, and a 150–300ms transition.
- On mobile, stack configuration fields and keep the primary “Run now” action full-width.
